# Compiling and Formatting Compact Code

## Compiling

The Compact CLI invokes the compiler through the `compile` subcommand. The CLI resolves the compiler binary from the artifact directory and passes arguments through to it.

### Basic Compilation

```bash
compact compile <source-path> <target-directory>
```

- `<source-path>`: Path to the `.compact` source file
- `<target-directory>`: Directory where output files are written

Output files include:
- TypeScript equivalent of the Compact source
- ZKIR circuit equivalents of exported circuits
- Proving keys generated by running ZKIR on the circuits

The proving keys and ZKIR circuits produced by compilation are consumed by the proof server at runtime to generate zero-knowledge proofs for transactions. See the **proof-server** skill for details on running and managing proof servers.

### Version-Specific Compilation

Prefix the compile command with `+VERSION` to use a specific compiler version instead of the default:

```bash
compact compile +0.25.0 src/contract.compact build/
```

The specified version must already be installed locally. Install it first with `compact update 0.25.0` if needed.

### Compiler Flags

These flags are passed through to the compiler binary (`compactc.bin`):

| Flag | Purpose |
|------|---------|
| `--help` | Print compiler help and exit |
| `--version` | Print compiler version and exit |
| `--language-version` | Print the Compact language version and exit |
| `--skip-zk` | Skip proving key generation (faster, useful during development) |
| `--no-communications-commitment` | Omit contract communications commitment for contract-to-contract calls |
| `--vscode` | Format errors for VS Code extension rendering |
| `--sourceRoot <path>` | Override the sourceRoot field in generated source maps |
| `--trace-passes` | Print compiler tracing information (for compiler developers) |

### Common Compilation Patterns

```bash
# Standard compilation
compact compile src/contract.compact build/

# Fast development iteration (skip ZK proof generation)
compact compile src/contract.compact build/ --skip-zk

# Check compiler version before compiling
compact compile --version

# Compile using a project-local toolchain
compact --directory ./.compact compile src/contract.compact build/
```

### Compilation with Custom Directory

When using `--directory` or `COMPACT_DIRECTORY`, the `--directory` flag must come before the `compile` subcommand:

```bash
# Correct: --directory before compile
compact --directory ./.compact compile src/contract.compact build/

# Incorrect: --directory after compile (will be passed to compiler as unknown flag)
compact compile --directory ./.compact src/contract.compact build/
```

## Formatting

The `compact format` command formats Compact source files in place, enforcing consistent code style.

### Basic Usage

```bash
# Format all .compact files in current directory and subdirectories
compact format

# Format specific files
compact format src/contract.compact src/lib.compact

# Format files in a specific directory
compact format src/
```

When no files are specified, `compact format` defaults to formatting all `.compact` files in the current directory (`.`), searching recursively.

### Check Mode

Use `--check` to verify formatting without modifying files:

```bash
compact format --check
```

Exit codes:
- `0`: All files are correctly formatted
- Non-zero: One or more files need formatting

This is useful for CI pipelines and pre-commit hooks to enforce formatting standards without modifying files.

### Verbose Output

Use `--verbose` (or `-v`) to see each file the formatter processes:

```bash
compact format --verbose
```

### Combining Flags

```bash
# Check formatting with verbose output
compact format --check --verbose

# Format files in a custom directory context
compact --directory ./.compact format src/
```

### CI Integration Example

In a CI pipeline, use `compact format --check` to fail the build if code is not formatted:

```yaml
# GitHub Actions example
- name: Check Compact formatting
  run: compact format --check
```

### Pre-commit Hook Example

```bash
#!/bin/bash
# .git/hooks/pre-commit
if ! compact format --check; then
  echo "Compact files need formatting. Run 'compact format' first."
  exit 1
fi
```
